# Brand Guide

One page. Read it, internalize it, then write everything else against it.

## What this project is

`job-search-agent` is an open-source local-first agent that scores job listings against the operator's full credentials, every day, automatically. It is a tool for builders who would rather ship one thoughtful application than spray fifty.

## Voice

**Confident, technical, builder-first.**

- Write the way a good staff engineer talks: precise, specific, no hedging, no exclamation marks.
- Assume the reader is competent. Skip the "What is a job board?" intro.
- Show, don't promise. Code samples and screenshots beat adjectives.
- Use the second person sparingly. Most copy can be written as a statement of fact about the tool, not a pitch at the reader.

**NOT: hype-y, cute, generic-startup-marketing-speak.**

- No "supercharge," "unleash," "revolutionize," "10x," "AI-powered" (it's table stakes — name what it does instead).
- No emojis in product copy. README badge row is the only exception.
- No "we're on a mission to..." If the README needs a mission statement, the README isn't ready.
- No screaming caps in product UI. Sentence case for everything except acronyms.

## Tagline

Primary, locked in:

> **Local-first job scout for the AI-native operator era.**

Acceptable secondary lines for specific surfaces:

- README hero: "Daily job scoring for your full credentials."
- Conversion CTA: "Stop spray-applying. Start scoring."
- Pitch one-liner: "A local agent that reads every new posting and ranks the ones worth your week."

## Headlines — pass vs fail

**Pass**
- "Score every new listing against your resume, locally, every morning."
- "Stop reading job boards. Read your ranked top five."
- "Your credentials, every posting, one number."
- "Built for operators who'd rather ship one strong application than fifty weak ones."

**Fail** (and why)
- "Supercharge your job search with AI!" — hype-y, exclamation, "AI" as a feature instead of a means.
- "Find your dream job today" — generic, vague, sounds like a 2014 jobs SaaS.
- "Discover hidden opportunities you'd never find on your own" — patronizing, vague, "discover" is filler.
- "We believe everyone deserves a great career" — mission-statement vibes, says nothing about the product.
- "The smartest way to apply" — comparative without a referent, marketing-speak.

## Words we like

scout · score · rank · listings · credentials · operator · pipeline · local-first · agent · scrape · digest · pin · shortlist · brief · signal-to-noise

## Words we avoid

unleash · supercharge · revolutionize · seamless · cutting-edge · dream job · journey · effortlessly · empower · best-in-class · synergy · "AI-powered" (as a standalone claim)

## Capitalization

- The project name is `job-search-agent`, lowercase, hyphenated, in mono. Never `Job Search Agent`, never `JobSearchAgent`, never `JSA`.
- Sentence case for all headlines and UI labels: "Today's top picks," not "Today's Top Picks."
- File names, env vars, and CLI flags follow each ecosystem's convention; don't force brand casing onto them.

## Imagery

- Monochrome screenshots and diagrams whenever possible. One accent color hit (`#10B981`) for the thing the eye should land on.
- Real terminal output > glossy product shots. The audience trusts logs.
- No stock photography of "diverse people at laptops." Ever.

## In one line

If a piece of copy could appear unchanged on a Series-A SaaS landing page, it's wrong. Rewrite it until it could only belong to `job-search-agent`.
