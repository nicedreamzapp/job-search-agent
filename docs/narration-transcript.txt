## Production notes

- Why we switched from VIDEO_PLAN.md §5 to SYNC_TABLE.md: §5 carries the pre-2026-05-21 single-terminal-scene narration (10 beats packed into a 58 s loop). The 12-beat / 60 s SYNC_TABLE supersedes it (see `SYNC_TABLE.md` §"Drift remediation log"). VIDEO_PLAN.md §5 should be re-synced to the new table on the next docs pass.
- Why Samantha (base) instead of "Enhanced": macOS shipped without the optional Enhanced voice packs on this machine. Base Samantha is still the cleanest neutral US-English option in `say`'s catalog for technical narration. To upgrade quality on a re-render, install the Enhanced voice pack from **System Settings → Accessibility → Spoken Content → System Voice → Manage Voices…** and re-run with `say -v "Samantha (Enhanced)"`.
- Per-beat AIFFs were rendered individually, converted to 48 kHz mono WAV, then mixed onto a single timeline using `ffmpeg adelay` + `amix` so each beat lands at its calibrated start offset. This avoids cumulative pacing drift that affects single-pass `say` renders.
- Loudness normalized in a second pass: `loudnorm=I=-16:TP=-1.5:LRA=11`.
